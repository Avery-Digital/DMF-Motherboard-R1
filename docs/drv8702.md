# DRV8702DQRHBRQ1 TEC H-Bridge Driver

## Overview

Three DRV8702DQRHBRQ1 H-bridge motor drivers control TEC (thermoelectric cooler) elements. Each instance has independent GPIO pins for direction/enable control and a unique SPI chip select for register access. All three share SPI2 with the LTC2338-18 ADC.

The driver operates in **PH/EN mode** (MODE pin LOW):
- **PH (IN1):** Sets current direction through the TEC
- **EN (IN2):** Enables the bridge — can accept PWM for power control

## Instance Pin Assignments

| Signal | Instance 1 | Instance 2 | Instance 3 |
|--------|------------|------------|------------|
| IN1/PH (direction) | PE9 (Pin 69) | PE13 | PJ8 |
| IN2/EN (enable) | PE11 (Pin 73) | PE14 | PJ10 |
| nSLEEP | PG5 (Pin 115) | PF0 | PF12 |
| MODE | PG6 (Pin 116) | PF1 | PF13 |
| nSCS (SPI CS) | PD1 (Pin 144) | PD0 | PD6 |
| nFAULT (input) | PG7 (Pin 117) | PF2 | PF14 |

All output pins are push-pull. nFAULT pins are inputs with internal pull-up (nFAULT is open-drain active-low from the DRV8702).

**Note:** PE9 and PE11 (Instance 1) are also TIM1_CH1/CH2 (AF1) — can be repurposed for PWM duty cycle control in a future revision.

## Initialization

`DRV8702_Init()` configures all six GPIO pins and sets safe defaults:

| Pin | Default State | Meaning |
|-----|---------------|---------|
| PH | LOW | Forward direction pre-selected |
| EN | LOW | Bridge outputs disabled |
| MODE | LOW | PH/EN control mode |
| nSCS | HIGH | SPI chip select deasserted |
| nSLEEP | LOW | Device in sleep mode |

After `DRV8702_Init()`, call `DRV8702_Wake()` to assert nSLEEP HIGH and bring the device active.

The init sequence in `main.c` initializes and wakes all three instances:

```c
DRV8702_Init(&drv8702_1_handle);  DRV8702_Wake(&drv8702_1_handle);
DRV8702_Init(&drv8702_2_handle);  DRV8702_Wake(&drv8702_2_handle);
DRV8702_Init(&drv8702_3_handle);  DRV8702_Wake(&drv8702_3_handle);
```

Error codes: `0x30` (instance 1), `0x31` (instance 2), `0x32` (instance 3).

## GPIO Control API

These functions are safe to call at any time after init — they do not use SPI.

### Direction and Enable

```c
DRV8702_SetDirection(handle, DRV8702_DIR_FORWARD);  // PH HIGH
DRV8702_SetDirection(handle, DRV8702_DIR_REVERSE);  // PH LOW
DRV8702_Enable(handle);                              // EN HIGH — current flows
DRV8702_Disable(handle);                             // EN LOW  — bridge coast
```

### Sleep / Wake

```c
DRV8702_Wake(handle);   // nSLEEP HIGH — device active
DRV8702_Sleep(handle);  // EN LOW first, then nSLEEP LOW — low-power sleep
```

### Mode Select

```c
DRV8702_SetMode(handle, DRV8702_MODE_PHEN);  // MODE LOW  — PH/EN control
DRV8702_SetMode(handle, DRV8702_MODE_PWM);   // MODE HIGH — PWM input mode
```

### Fault Check

```c
bool fault = DRV8702_IsFaulted(handle);  // Reads nFAULT pin (LOW = fault)
```

## TEC Convenience Functions

These wrap direction + enable with a fault pre-check:

```c
DRV8702_TEC_Heat(handle);   // Check nFAULT, set FORWARD, enable
DRV8702_TEC_Cool(handle);   // Check nFAULT, set REVERSE, enable
DRV8702_TEC_Stop(handle);   // Disable bridge (coast)
```

Returns `DRV8702_ERR_FAULT` if nFAULT is asserted, `DRV8702_ERR_NOT_INIT` if not initialized, `DRV8702_OK` on success.

**Important:** Which direction (FORWARD/REVERSE) actually heats vs cools the TEC depends on PCB wiring. Verify against the schematic before use in production.

## SPI Register Access

The DRV8702 provides SPI-based fault status readback and configuration. SPI2 is shared with the LTC2338-18 ADC (32-bit), so the driver temporarily switches to 16-bit frames for each transfer.

### SPI Frame Format (16-bit, MSB first, Mode 0)

```
Bit [15]:     R/W  — 0 = write, 1 = read
Bits [14:11]: ADDR — 4-bit register address
Bits [10:0]:  DATA — 11-bit register data
```

### Register Map

| Register | Address | R/W | Description |
|----------|---------|-----|-------------|
| IC_STAT | 0x00 | R | IC fault status |
| VGS_STAT | 0x01 | R | VGS fault status |
| IC_CTRL | 0x02 | R/W | IC control (CLR_FLT) |
| DRIVE_CTRL | 0x03 | R/W | Drive control |
| GATE_HS | 0x04 | R/W | Gate drive high-side |
| GATE_LS | 0x05 | R/W | Gate drive low-side |
| OCP_CTRL | 0x06 | R/W | Overcurrent protection |
| CSA_CTRL | 0x07 | R/W | Current sense amplifier |

**Register map notice:** All addresses and bit fields are derived from TI DRV8702D documentation (SLVSDF9). Verify against the actual datasheet before using SPI register access in production.

### IC_STAT (0x00) Bit Flags

| Bit | Name | Description |
|-----|------|-------------|
| 10 | FAULT | Any fault is latched |
| 9 | OCP | Overcurrent protection tripped |
| 8 | OTSD | Overtemperature shutdown |
| 7 | UVLO | Undervoltage lockout |
| 6 | OTW | Overtemperature warning |

### SPI Register API

```c
DRV8702_WriteReg(handle, addr, data);     // Write 11-bit data to register
DRV8702_ReadReg(handle, addr, &data_out); // Read 11-bit data from register
DRV8702_ReadFaultStatus(handle);          // Read IC_STAT, cache in handle
DRV8702_ClearFaults(handle);              // Write CLR_FLT to IC_CTRL
```

### SPI Transfer Sequence

1. Disable SPI2, set data width to 16-bit, re-enable
2. Assert nSCS LOW
3. Flush stale RX data
4. Transmit 16-bit frame, wait for RXP
5. Read 16-bit response
6. Deassert nSCS HIGH
7. Disable SPI2, restore 32-bit data width, re-enable

Timeout: 5 ms (`DRV8702_SPI_TIMEOUT_MS`).

## Return Codes

```c
typedef enum {
    DRV8702_OK           = 0,   /* Success */
    DRV8702_ERR_NOT_INIT = 1,   /* Handle not initialised */
    DRV8702_ERR_FAULT    = 2,   /* nFAULT asserted */
    DRV8702_ERR_TIMEOUT  = 3,   /* SPI transfer timeout */
    DRV8702_ERR_SPI      = 4,   /* SPI bus error */
} DRV8702_Status;
```

## Handle State

Each `DRV8702_Handle` tracks:

| Field | Type | Description |
|-------|------|-------------|
| `cfg` | `const DRV8702_Config *` | Pin and SPI config (flash) |
| `initialised` | `bool` | Set true by `DRV8702_Init()` |
| `faulted` | `bool` | Updated by `IsFaulted()` and `ReadFaultStatus()` |
| `last_fault_reg` | `uint16_t` | Last IC_STAT register value read via SPI |

## Concurrency

- GPIO control functions (direction, enable, sleep) are safe to call at any time
- SPI register functions must not be called concurrently with `SPI_LTC2338_Read()`
- No RTOS means all access is currently sequential and safe
