# ADS7066IRTER — 8-Channel 16-bit 250-kSPS SAR ADC

## Overview

The ADS7066IRTER is an 8-channel, 16-bit, 250 kSPS SAR ADC with an integrated 2.5V reference, programmable averaging, and configurable GPIO channels. Three instances share SPI2 with the LTC2338-18 ADC, DRV8702 TEC drivers, and DAC80508 DAC.

Each channel can be individually configured as an analog input, digital input, or digital output.

Reference: TI ADS7066 datasheet (SBAS928C)

## Instance Pin Assignments

| Instance | nCS Pin | nCS Port | Pin # |
|----------|---------|----------|-------|
| 1 | PD5 | GPIOD | 148 |
| 2 | PD4 | GPIOD | 147 |
| 3 | PD3 | GPIOD | 146 |

All three share SPI2 bus signals (PC2 MISO, PC3 MOSI, PA9 SCK).

PD3–PD5 are configured by each ADS7066 instance's `ADS7066_Init()` via `Pin_Init()` and driven HIGH (deasserted).

## SPI Configuration

| Parameter | Value | Notes |
|-----------|-------|-------|
| Register frames | 24-bit | [opcode(8)] [address(8)] [data(8)] |
| Data read frames | 16-bit | 16-bit ADC result, MSB first |
| SPI Mode | 0 (CPOL=0, CPHA=0) | Default, matches SPI2 — no CPHA switch needed |
| Max SCLK | 60 MHz | We run at 16 MHz (PLL3P 128 MHz / DIV8) |
| Timeout | 5 ms | `ADS7066_SPI_TIMEOUT_MS` |

The ADS7066 supports all 4 SPI modes via the `CPOL_CPHA` register field, but defaults to Mode 0 on power-up. This driver uses the default.

## SPI Bus Sharing

SPI2 now hosts 5 device types:

| Device | Data Width | SPI Mode | Notes |
|--------|-----------|----------|-------|
| LTC2338-18 (ADC) | 32-bit | Mode 0 | Default SPI2 config |
| DRV8702 x3 (TEC) | 16-bit | Mode 0 | Switches data width |
| DAC80508 (DAC) | 24-bit | Mode 1 | Switches data width + CPHA |
| ADS7066 x3 (ADC) | 24-bit / 16-bit | Mode 0 | Switches data width only |

The ADS7066 driver uses two transfer functions:
- `ADS7066_SPITransfer24()` — 24-bit Mode 0 for register read/write
- `ADS7066_SPITransfer16()` — 16-bit Mode 0 for ADC data reads

Both reconfigure data width before the transfer and restore 32-bit on exit.

## SPI Opcodes

| Opcode | Value | Description |
|--------|-------|-------------|
| NOOP | `0x00` | No operation |
| READ | `0x10` | Single register read |
| WRITE | `0x08` | Single register write |
| SET_BIT | `0x18` | Set specified bits at address |
| CLR_BIT | `0x20` | Clear specified bits at address |

## SPI Frame Format (24-bit)

```
Bits [23:16]: Opcode (8 bits)
Bits [15:8]:  Register address (8 bits)
Bits [7:0]:   Data (8 bits)
```

**Register read** requires two SPI frames:
1. Send `[READ_OP] [address] [0x00]` — decoded on CS rising edge
2. Send `[NOOP] [0x00] [0x00]` — response bits [23:16] contain the register data

## Conversion Pipeline

The ADS7066 starts a conversion on the **CS rising edge**. The result is available on SDO during the **next** CS-low frame.

In manual mode, after a channel switch via `MANUAL_CHID`:
- **Cycle N:** Register write decoded on CS rise → MUX switches to new channel
- **Cycle N+1:** ADC samples new channel, outputs **previous** channel's data
- **Cycle N+2:** ADC outputs **new** channel's data

`ADS7066_ReadChannel()` handles this automatically by performing dummy reads after a channel switch.

```
              Cycle N              Cycle N+1            Cycle N+2
CS      ──┐     ┌────┐     ┌────┐     ┌────┐     ┌──
          └─────┘    └─────┘    └─────┘    └─────┘
SDI     [WR CHID=Y]  [NOOP]     [NOOP]     [NOOP]
SDO      (old data)  (old data) (AINx data) (AINy data)
MUX      → AINx      → AINy     → AINy      → AINy
```

## Register Map

| Address | Name | Reset | Description |
|---------|------|-------|-------------|
| 0x00 | SYSTEM_STATUS | 0x81 | BOR flag, CRC error status |
| 0x01 | GENERAL_CFG | 0x00 | REF_EN, CRC_EN, RANGE, CAL, RST |
| 0x02 | DATA_CFG | 0x00 | FIX_PAT, APPEND_STATUS, CPOL_CPHA |
| 0x03 | OSR_CFG | 0x00 | Oversampling ratio (2x to 128x) |
| 0x04 | OPMODE_CFG | 0x04 | Oscillator, clock divider |
| 0x05 | PIN_CFG | 0x00 | Per-channel AIN vs GPIO selection |
| 0x07 | GPIO_CFG | 0x00 | GPIO direction (input/output) |
| 0x09 | GPO_DRIVE_CFG | 0x00 | GPIO output type (open-drain/push-pull) |
| 0x0B | GPO_VALUE | 0x00 | GPIO output levels |
| 0x0D | GPI_VALUE | 0x00 | GPIO input readback |
| 0x10 | SEQUENCE_CFG | 0x00 | SEQ_MODE, SEQ_START |
| 0x11 | CHANNEL_SEL | 0x00 | MANUAL_CHID[3:0] |
| 0x12 | AUTO_SEQ_CHSEL | 0x00 | Auto-sequence channel enable mask |

## Key Register Fields

### GENERAL_CFG (0x01)

| Bit | Field | Description |
|-----|-------|-------------|
| 7 | REF_EN | 1 = Enable internal 2.5V reference |
| 6 | CRC_EN | 1 = Enable CRC on SPI interface |
| 3 | RANGE | 0 = 0–VREF, 1 = 0–2xVREF |
| 1 | CAL | 1 = Trigger offset calibration (auto-clears) |
| 0 | RST | 1 = Software reset (auto-clears after ~5 ms) |

### SEQUENCE_CFG (0x10)

| Bit | Field | Description |
|-----|-------|-------------|
| 4 | SEQ_START | 1 = Start auto-sequence scanning |
| 1:0 | SEQ_MODE | 00 = Manual, 01 = Auto-sequence, 10 = On-the-fly |

## Functional Modes

| Mode | Conversion Control | MUX Control |
|------|-------------------|-------------|
| Manual | CS rising edge | Register write to MANUAL_CHID |
| On-the-fly | CS rising edge | First 5 SDI bits after CS falling edge |
| Auto-sequence | CS rising edge | Channel sequencer (AUTO_SEQ_CHSEL) |
| Autonomous | Internal | Channel sequencer |

The driver defaults to **manual mode** (power-up default).

## API Reference

### `ADS7066_Init(ADS7066_Handle *handle)`

Initialize nCS pin, clear BOR flag. Error codes: `0x50` (inst 1), `0x51` (inst 2), `0x52` (inst 3).

### `ADS7066_WriteReg(handle, addr, data)`

Write 8-bit data to a register via 24-bit SPI frame.

### `ADS7066_ReadReg(handle, addr, &data_out)`

Read 8-bit data from a register (two SPI frames).

### `ADS7066_SelectChannel(handle, channel)`

Write `MANUAL_CHID` to select an analog input (0–7) for the next conversion.

### `ADS7066_ReadConversion(handle, &result_out)`

Read the 16-bit result from the previous conversion via a 16-bit SPI frame. The CS rising edge triggers a new conversion.

### `ADS7066_ReadChannel(handle, channel, &result_out)`

Convenience function: selects a channel (if different from current), handles the 2-cycle pipeline latency with dummy reads, then returns the valid 16-bit result.

### `ADS7066_SoftReset(handle)`

Software reset. All registers return to defaults. Device needs ~5 ms to complete.

### `ADS7066_EnableInternalRef(handle)`

Enable the internal 2.5V reference (disabled by default on power-up).

### `ADS7066_Calibrate(handle)`

Trigger ADC offset calibration. The CAL bit auto-clears when complete.

## Return Codes

```c
typedef enum {
    ADS7066_OK           = 0,   /* Success */
    ADS7066_ERR_NOT_INIT = 1,   /* Handle not initialised */
    ADS7066_ERR_TIMEOUT  = 2,   /* SPI transfer timeout */
    ADS7066_ERR_SPI      = 3,   /* SPI bus error */
    ADS7066_ERR_PARAM    = 4,   /* Invalid parameter */
} ADS7066_Status;
```

## ADC Transfer Function

```
Output Code = (VIN / FSR) x 65536
```

Where:
- `FSR` = VREF (RANGE=0) or 2 x VREF (RANGE=1)
- `VREF` = 2.5V internal or external reference
- Output range: 0x0000 (0V) to 0xFFFF (FSR - 1 LSB)

## Programmable Averaging

The OSR_CFG register enables oversampling with averaging:

| OSR[2:0] | Samples | Output Resolution |
|----------|---------|-------------------|
| 000 | 1 (none) | 16-bit |
| 001 | 2 | 17-bit |
| 010 | 4 | 18-bit |
| 011 | 8 | 19-bit |
| 100 | 16 | 20-bit |
| 101 | 32 | 20-bit |
| 110 | 64 | 20-bit |
| 111 | 128 | 20-bit |

When averaging is enabled, the output is 20 bits (MSB-aligned in a 24-bit SPI frame).
